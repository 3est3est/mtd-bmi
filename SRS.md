# Software Requirements Specification (SRS)
## สำหรับโครงการ Web Application คำนวณและติดตามค่าดัชนีมวลกาย (BMI)

### 1. บทนำ (Introduction)
เอกสารฉบับนี้จัดทำขึ้นเพื่อกำหนดความต้องการและขอบเขตของการพัฒนา Web Application สำหรับคำนวณและติดตามค่าดัชนีมวลกาย (BMI) โดยมุ่งเน้นการใช้งานที่ง่าย รองรับผู้ใช้งานหลายคน และมีระบบรายงานที่ครอบคลุม

### 2. รายละเอียดโดยรวม (Overall Description)
แอปพลิเคชันนี้เป็นระบบเว็บที่ช่วยให้ผู้ใช้งานสามารถบันทึกส่วนสูงและน้ำหนักเพื่อคำนวณค่า BMI และติดตามแนวโน้มสุขภาพของตนเองได้

#### 2.1 คุณลักษณะของผู้ใช้งาน (User Classes and Characteristics)
- **ผู้ใช้งานทั่วไป (General User):** สามารถสมัครสมาชิก เข้าสู่ระบบ บันทึกข้อมูล BMI ของตนเอง และดูรายงานส่วนตัวได้
- **ผู้ดูแลระบบ (Admin):** สามารถดูภาพรวมข้อมูลและรายงาน MIS ในระดับภาพรวมได้ (ถ้ามี)

#### 2.2 สภาพแวดล้อมการทำงาน (Operating Environment)
- **Frontend/Backend:** Next.js (Latest version)
- **Database:** SQLite (ผ่าน Prisma หรือ Drizzle ORM)
- **UI Framework:** Tailwind CSS พร้อมการตกแต่งสไตล์ **Catppuccin Mocha**

### 3. ความต้องการด้านฟังก์ชัน (System Features)

#### 3.1 การจัดการผู้ใช้งาน (User Management)
- ระบบสมัครสมาชิก (Registration)
- ระบบเข้าสู่ระบบ (Authentication) โดยรองรับผู้ใช้งานหลายคน (Multi-user)
- ข้อมูลส่วนตัวพื้นฐาน (ชื่อ, อายุ, เพศ)

#### 3.2 การคำนวณและบันทึก BMI (BMI Calculation & Tracking)
- คำนวณค่า BMI จากน้ำหนัก (kg) และส่วนสูง (cm)
- บันทึกประวัติการคำนวณลงในฐานข้อมูลพร้อมวันที่และเวลา
- แสดงผลลัพธ์พร้อมคำแนะนำตามเกณฑ์มาตรฐาน

#### 3.3 ระบบรายงาน MIS (MIS Reporting)
ระบบต้องสามารถแสดงรายงานสรุปผลย้อนหลังได้ในรูปแบบต่างๆ ดังนี้:
- **รายงานรายวัน (Daily Report):** แสดงข้อมูลสรุปการบันทึกในแต่ละวัน
- **รายงานรายสัปดาห์ (Weekly Report):** แสดงแนวโน้มและการเปลี่ยนแปลงในรอบสัปดาห์
- **รายงานรายเดือน (Monthly Report):** สรุปผลรายเดือนเพื่อดูความคืบหน้าในระยะกลาง
- **รายงานรายปี (Yearly Report):** แสดงภาพรวมการเปลี่ยนแปลงของสุขภาพในรอบปี

### 4. ความต้องการด้านการออกแบบและเทคนิค (Design and Technical Requirements)

#### 4.1 UI/UX Design
- ใช้ธีมสี **Catppuccin Mocha** (Dark theme palette)
- การออกแบบที่เน้นความเรียบหรู (Minimalist) และใช้งานง่าย (User-friendly)
- รองรับการแสดงผลบนมือถือ (Responsive Design)

#### 4.2 Database Schema (SQLite)
- **Users Table:** เก็บข้อมูลผู้ใช้งาน (id, email, password, profile)
- **BMILogs Table:** เก็บประวัติการบันทึก (id, user_id, weight, height, bmi_value, timestamp)

### 5. ความต้องการอื่นๆ (Other Requirements)
- ความปลอดภัยของข้อมูลผู้ใช้งาน
- ประสิทธิภาพในการโหลดหน้าเว็บที่รวดเร็ว
- ความถูกต้องแม่นยำของสูตรการคำนวณ BMI
