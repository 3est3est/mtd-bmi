# Development Task List: BMI Web Application

แผนการพัฒนาแอปพลิเคชันคำนวณ BMI โดยแบ่งออกเป็นเฟสต่างๆ เพื่อให้ง่ายต่อการติดตามความคืบหน้า

## เฟส 1: การตั้งค่าโครงการและโครงสร้างพื้นฐาน (Project Setup)
- [x] 1.1 เริ่มต้นโครงการ Next.js (App Router)
- [x] 1.2 ติดตั้ง Tailwind CSS และกำหนดค่าสีธีม **Catppuccin Mocha**
- [x] 1.3 ตั้งค่าฐานข้อมูล SQLite (ใช้ Prisma หรือ Drizzle ORM)
- [x] 1.4 สร้าง Schema พื้นฐานสำหรับ User และ BMILogs
- [x] 1.5 สร้างโครงสร้างโฟลเดอร์ (Components, Lib, Actions, Types)

## เฟส 2: ระบบจัดการผู้ใช้งาน (Authentication)
- [x] 2.1 ติดตั้งและตั้งค่า NextAuth.js หรือระบบ Auth พื้นฐาน
- [x] 2.2 พัฒนาหน้าสมัครสมาชิก (Register)
- [x] 2.3 พัฒนาหน้าเข้าสู่ระบบ (Login)
- [x] 2.4 จัดการ Session และการป้องกัน Route (Protected Routes)

## เฟส 3: ฟังก์ชันหลักและการบันทึกข้อมูล (Core Features)
- [x] 3.1 พัฒนาหน้าคำนวณ BMI (Input Weight/Height)
- [x] 3.2 ระบบบันทึกข้อมูลลง SQLite พร้อมระบุ User ID
- [x] 3.3 แสดงผลลัพธ์และระดับเกณฑ์สุขภาพหลังการคำนวณ
- [x] 3.4 พัฒนาหน้าประวัติการบันทึก (History List)

## เฟส 4: ระบบรายงาน MIS (Reporting System)
- [x] 4.1 พัฒนา Logic สำหรับดึงข้อมูลสรุปรายวัน (Daily)
- [x] 4.2 พัฒนา Logic สำหรับดึงข้อมูลสรุปรายสัปดาห์ (Weekly)
- [x] 4.3 พัฒนา Logic สำหรับดึงข้อมูลสรุปรายเดือน (Monthly)
- [x] 4.4 พัฒนา Logic สำหรับดึงข้อมูลสรุปรายปี (Yearly)
- [x] 4.5 แสดงผลรายงานในรูปแบบกราฟหรือตารางสรุป (ใช้ Recharts หรือ Chart.js)

## เฟส 5: การปรับแต่ง UI/UX และการตรวจสอบ (Polishing & Testing)
- [x] 5.1 ตกแต่ง UI ทั้งหมดให้เป็นสไตล์ **Catppuccin Mocha** อย่างสมบูรณ์
- [x] 5.2 ทดสอบการใช้งาน Responsive บนอุปกรณ์ต่างๆ
- [x] 5.3 ตรวจสอบความถูกต้องของสูตรคำนวณและข้อมูลในรายงาน
- [x] 5.4 จัดการ Error Handling และ Loading States
